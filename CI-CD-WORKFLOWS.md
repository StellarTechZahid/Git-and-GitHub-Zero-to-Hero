# Production-Ready CI/CD Workflows 🚀\n\n## Complete GitHub Actions Workflow Example\n\n### 1. Basic CI/CD Pipeline\n\n```yaml\nname: CI/CD Pipeline\n\non:\n  push:\n    branches: [ main, develop ]\n  pull_request:\n    branches: [ main ]\n\njobs:\n  test:\n    runs-on: ubuntu-latest\n    steps:\n      - uses: actions/checkout@v3\n      \n      - name: Setup Node.js\n        uses: actions/setup-node@v3\n        with:\n          node-version: '18'\n      \n      - name: Install dependencies\n        run: npm ci\n      \n      - name: Lint code\n        run: npm run lint\n      \n      - name: Run tests\n        run: npm test -- --coverage\n      \n      - name: Upload coverage\n        uses: codecov/codecov-action@v3\n        with:\n          files: ./coverage/coverage-final.json\n\n  build:\n    needs: test\n    runs-on: ubuntu-latest\n    steps:\n      - uses: actions/checkout@v3\n      \n      - name: Build Docker image\n        run: docker build -t myapp:${{ github.sha }} .\n      \n      - name: Login to Docker Hub\n        uses: docker/login-action@v2\n        with:\n          username: ${{ secrets.DOCKER_USERNAME }}\n          password: ${{ secrets.DOCKER_PASSWORD }}\n      \n      - name: Push image\n        run: docker push myapp:${{ github.sha }}\n\n  deploy:\n    needs: build\n    runs-on: ubuntu-latest\n    if: github.ref == 'refs/heads/main'\n    steps:\n      - uses: actions/checkout@v3\n      \n      - name: Deploy to production\n        run: |\n          echo \"Deploying to production...\"\n          # Add deployment commands\n```\n\n### 2. Multi-Environment Deployment\n\n```yaml\nname: Multi-Environment Deploy\n\non:\n  push:\n    branches: [ main, staging ]\n\njobs:\n  deploy:\n    runs-on: ubuntu-latest\n    strategy:\n      matrix:\n        environment: [staging, production]\n      max-parallel: 1\n    environment:\n      name: ${{ matrix.environment }}\n    steps:\n      - uses: actions/checkout@v3\n      \n      - name: Deploy to ${{ matrix.environment }}\n        env:\n          ENV_NAME: ${{ matrix.environment }}\n          DEPLOY_TOKEN: ${{ secrets[format('{0}_DEPLOY_TOKEN', matrix.environment)] }}\n        run: |\n          echo \"Deploying to $ENV_NAME\"\n          # Deploy commands here\n      \n      - name: Run smoke tests\n        run: npm run test:smoke -- --env=${{ matrix.environment }}\n      \n      - name: Notify Slack\n        uses: 8398a7/action-slack@v3\n        with:\n          status: ${{ job.status }}\n          text: 'Deployment to ${{ matrix.environment }} completed'\n        if: always()\n```\n\n### 3. Security Scanning Pipeline\n\n```yaml\nname: Security Scan\n\non:\n  push:\n    branches: [ main ]\n  pull_request:\n    branches: [ main ]\n\njobs:\n  security:\n    runs-on: ubuntu-latest\n    steps:\n      - uses: actions/checkout@v3\n      \n      - name: Run Trivy vulnerability scanner\n        uses: aquasecurity/trivy-action@master\n        with:\n          scan-type: 'fs'\n          scan-ref: '.'\n          format: 'sarif'\n          output: 'trivy-results.sarif'\n      \n      - name: Upload Trivy results to GitHub Security\n        uses: github/codeql-action/upload-sarif@v2\n        with:\n          sarif_file: 'trivy-results.sarif'\n      \n      - name: Check for secrets\n        uses: trufflesecurity/trufflehog@main\n        with:\n          path: ./\n          base: ${{ github.event.repository.default_branch }}\n      \n      - name: Dependency check\n        run: npm audit --audit-level=moderate\n```\n\n### 4. Release Workflow\n\n```yaml\nname: Release\n\non:\n  push:\n    tags:\n      - 'v*'\n\njobs:\n  release:\n    runs-on: ubuntu-latest\n    steps:\n      - uses: actions/checkout@v3\n      \n      - name: Create Release\n        uses: actions/create-release@v1\n        env:\n          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}\n        with:\n          tag_name: ${{ github.ref }}\n          release_name: Release ${{ github.ref }}\n          body: |\n            Changes in this Release:\n            - Automated release\n            - Check commit log for details\n          draft: false\n          prerelease: false\n      \n      - name: Build and publish packages\n        run: |\n          npm run build\n          npm publish\n```\n\n### 5. Nightly Build & Regression Testing\n\n```yaml\nname: Nightly Build\n\non:\n  schedule:\n    - cron: '0 2 * * *'  # 2 AM UTC daily\n  workflow_dispatch:\n\njobs:\n  nightly-tests:\n    runs-on: ubuntu-latest\n    steps:\n      - uses: actions/checkout@v3\n      \n      - name: Run comprehensive tests\n        run: |\n          npm install\n          npm run test:full\n          npm run test:e2e\n          npm run test:performance\n      \n      - name: Generate report\n        run: npm run report:generate\n      \n      - name: Upload results\n        uses: actions/upload-artifact@v3\n        with:\n          name: nightly-results\n          path: ./reports/\n      \n      - name: Notify on failure\n        if: failure()\n        uses: 8398a7/action-slack@v3\n        with:\n          status: 'failure'\n          text: 'Nightly build failed'\n          webhook_url: ${{ secrets.SLACK_WEBHOOK }}\n```\n\n---\n\n## GitOps Workflow with ArgoCD\n\n```yaml\n# argocd-application.yaml\napiVersion: argoproj.io/v1alpha1\nkind: Application\nmetadata:\n  name: myapp\n  namespace: argocd\nspec:\n  project: default\n  source:\n    repoURL: https://github.com/StellarTechZahid/Git-and-GitHub-Zero-to-Hero.git\n    targetRevision: main\n    path: k8s/\n  destination:\n    server: https://kubernetes.default.svc\n    namespace: production\n  syncPolicy:\n    automated:\n      prune: true\n      selfHeal: true\n    syncOptions:\n    - CreateNamespace=true\n```\n\n---\n\n## Best Practices\n\n### 1. Error Handling\n```\n✓ Add retry logic for flaky tests\n✓ Notify team on failures\n✓ Implement rollback mechanism\n✓ Document error resolution\n✓ Monitor workflow performance\n```\n\n### 4. Documentation\n```\n✓ Document workflow logic\n✓ Include deployment procedures\n✓ Maintain runbooks\n✓ Track workflow changes\n✓ Create incident procedures\n```\n\n---\n\n## Testing Strategy\n\n```\nUnit Tests\n    ↓\nIntegration Tests\n    ↓\nEnd-to-End Tests\n    ↓\nPerformance Tests\n    ↓\nSecurity Tests\n    ↓\nSmoke Tests (Production)\n    ↓\nMonitoring & Alerts\n```\n\n---\n\n## Deployment Strategies\n\n### Blue-Green Deployment\n```\nBlue (Current)    Green (New)\n    ↓                 ↓\n  Live          Ready to switch\n    ↓                 ↓\n   80%               20%\n(traffic)          (traffic)\n    ↓ Quick rollback if needed\n   All traffic → Green\n```\n\n### Canary Deployment\n```\nStable:   95%\n   ↓\nCanary:    5%\n   ↓\nMonitor for issues\n   ↓\nGradually increase → 100%\n```\n\n### Rolling Deployment\n```\nVersion 1: ████░░░░░░\n              ↓ (rolling update)\nVersion 2: ░████░░░░░\n              ↓\nVersion 2: ░░████░░░░\n              ↓\nVersion 2: ░░░████░░░\n              ↓\nVersion 2: ██████████\n```\n\n---\n\n## Monitoring & Alerts\n\n```yaml\nMetrics to Monitor:\n├─ Deployment frequency\n├─ Lead time for changes\n├─ Mean time to recovery (MTTR)\n├─ Change failure rate\n├─ Uptime/Availability\n├─ Error rates\n├─ Response times\n└─ Resource utilization\n```\n\n---\n\n## Troubleshooting Common Issues\n\n### Workflow Fails Intermittently\n```\n1.
