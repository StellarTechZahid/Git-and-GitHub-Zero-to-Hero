```
     ↓

     ↓
Automated tests run
     ↓
Code quality checks
     ↓
Build Docker image
     ↓
Push to registry
     ↓
Deploy to staging
     ↓
Integration tests
     ↓
Approval required
     ↓
Deploy to production
     ↓
Monitoring and alerts
```

## GitOps Principles

1. **Declarative**: Describe desired state in Git
2. **Git as Source of Truth**: Everything tracked in Git
3. **Automated Delivery**: Changes apply automatically
4. **Audit Trail**: Complete history in Git
5. **Easy Rollback**: Revert by changing Git
